# 🛍️ Wildberries_Course_project

A collaborative project inspired by the Wildberries interface. This project is developed to practice modern frontend development with Vanilla JavaScript.

---

## 🚀 Quick Start

### Clone and install:

```bash
git clone https://github.com/martasusenkova/Wildberries_Course_project.git
cd Wildberries_Course_project
npm install
```

### Development:

```bash
npm run dev
```

---

## 📜 Available Scripts for Development

- `npm run dev` – start development server (Vite)
- `npm run lint` – run ESLint to check for code issues
- `npm run format` – format code with Prettier

---

## 🔄 Git Workflow

1. Create a new branch:

   ```bash
   git checkout -b feat/your-feature
   ```

2. Commit your changes:

   ```bash
   git commit -m "feat: add search input component"
   ```

3. Push the branch:

   ```bash
   git push origin feat/your-feature
   ```

4. Create a pull request to merge into the `develop` branch

5. Wait for approval before merging

---

## 📦 Conventional Commit Types

We use [Conventional Commits](https://www.conventionalcommits.org/) to keep a clean and readable git history:

- `feat` – new features or functionality
- `fix` – bug fixes
- `refactor` – code refactoring that doesn't change behavior
- `style` – formatting changes, no logic changes (e.g. semicolons, spacing)
- `build` – changes to the build system or dependencies
- `perf` – performance improvements
- `chore` – routine tasks or maintenance
- `revert` – reverting a previous commit
